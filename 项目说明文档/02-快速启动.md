# 快速启动

本文档给出最短可用启动流程。第一次接触项目时，建议先用 Docker Compose 跑通整体服务，再根据需要切换到本地开发模式。

## 环境要求

- JDK 17
- Node.js 18+ 或 20+
- npm
- MySQL 8.0
- Redis 7
- Docker 和 Docker Compose，可选但推荐
- Docker Compose 部署时会自动构建 H5 并启动 SillyTavern

## 方式一：Docker Compose 启动

1. 准备环境变量。

```bash
cd deploy
cp .env.example .env
```

2. 编辑 `deploy/.env`，至少替换以下值。

```text
MYSQL_ROOT_PASSWORD
MYSQL_PASSWORD
APP_AUTH_SECRET
APP_RUOYI_ADMIN_PASSWORD
APP_RUOYI_JWT_SECRET
SILLYTAVERN_PUBLIC_BASE_URL
```

3. 启动服务。

```bash
cd deploy
docker compose --env-file .env up -d --build
```

默认访问地址：

| 服务 | 地址 |
| --- | --- |
| 后端 API | `http://127.0.0.1:8080` |
| 后端健康检查 | `http://127.0.0.1:8080/actuator/health` |
| 管理后台 | `http://127.0.0.1:8081` |
| H5 客户端 | `http://127.0.0.1:8082` |
| SillyTavern | `http://127.0.0.1:8000` |

4. 查看运行状态。

```bash
docker compose ps
docker compose logs -f backend
```

## 方式二：本地开发启动

### 1. 启动依赖服务

准备 MySQL 8 和 Redis 7。数据库建议使用 `utf8mb4` 字符集。

可以参考根目录 `.env.example` 创建数据库：

```text
MYSQL_DATABASE=silly_app
MYSQL_USER=silly
MYSQL_PASSWORD=change-this-db-password
```

### 2. 启动后端

后端配置主要通过环境变量注入。PowerShell 示例：

```powershell
$env:APP_AUTH_SECRET="replace-with-long-random-secret"
$env:APP_RUOYI_ADMIN_USERNAME="admin"
$env:APP_RUOYI_ADMIN_PASSWORD="{bcrypt}replace-with-bcrypt-hash"
$env:APP_RUOYI_JWT_SECRET="replace-with-long-random-jwt-secret"
$env:SPRING_DATASOURCE_URL="jdbc:mysql://localhost:3306/silly_app?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai"
$env:SPRING_DATASOURCE_USERNAME="silly"
$env:SPRING_DATASOURCE_PASSWORD="change-this-db-password"
$env:SPRING_DATA_REDIS_HOST="localhost"
$env:SPRING_DATA_REDIS_PORT="6379"
```

启动：

```bash
cd backend
./mvnw spring-boot:run
```

Windows：

```powershell
cd backend
.\mvnw.cmd spring-boot:run
```

启动后访问：

```text
http://127.0.0.1:8080/actuator/health
```

### 3. 启动管理端

```bash
cd admin-web
npm install
npm run dev
```

开发环境样例配置位于 `admin-web/.env.development.example`。默认后端地址：

```text
VITE_BACKEND_URL=http://127.0.0.1:8080
```

### 4. 启动 H5 客户端

```bash
cd h5-web
npm install
npm run dev:h5
```

H5 本地开发默认后端地址主要在 `h5-web/common/api.js` 中维护；CLI 构建产物位于 `h5-web/dist/build/h5`。

## 常见启动问题

### 后端数据库连接失败

检查：

- MySQL 是否启动。
- 数据库名、用户名、密码是否和环境变量一致。
- JDBC URL 中时区和字符集是否正确。
- Docker 场景下容器内应使用 `mysql` 服务名，而不是宿主机的 `localhost`。

### 后端 Redis 连接失败

检查 Redis 是否启动，以及 `SPRING_DATA_REDIS_HOST`、`SPRING_DATA_REDIS_PORT`、`SPRING_DATA_REDIS_PASSWORD` 是否正确。

如果只是做部分本地测试，可以尝试使用 `local` profile，但完整聊天、限流、WebSocket 等能力仍建议准备 Redis。

### 管理端接口 404 或跨域

检查：

- `admin-web/.env.development` 的 `VITE_BACKEND_URL`。
- 后端 `APP_CORS_ALLOWED_ORIGIN_PATTERNS` 是否包含当前前端地址。
- Vite dev server 代理配置是否生效。

### H5 无法请求后端

检查：

- `h5-web/common/api.js` 中的本地或远程 API 地址。
- 浏览器控制台中的 CORS 报错。
- 后端是否允许对应 Origin。
- 如果走 nginx，同步检查 `h5-web/deploy/nginx.conf` 的代理路径。
