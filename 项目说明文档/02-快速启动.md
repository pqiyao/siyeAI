# 快速启动

这份文档只讲最短路径。

- 想先把整套系统跑起来：用 Docker Compose。
- 想自己改代码开发：用本地开发方式。

## 环境要求

- JDK 17
- Node.js 18+ 或 20+
- npm
- MySQL 8
- Redis 7
- Docker 和 Docker Compose（推荐）

## 方式一：Docker Compose

这是最推荐的方式，适合第一次体验项目。

### 1. 准备配置

```bash
cd deploy
cp .env.example .env
```

至少把下面这些值改掉：

```text
MYSQL_ROOT_PASSWORD
MYSQL_PASSWORD
APP_AUTH_SECRET
APP_RUOYI_ADMIN_PASSWORD
APP_RUOYI_JWT_SECRET
SILLYTAVERN_PUBLIC_BASE_URL
```

### 2. 启动

```bash
cd deploy
docker compose --env-file .env up -d --build
```

### 3. 访问地址

| 服务 | 地址 |
| --- | --- |
| 后端 API | `http://127.0.0.1:8080` |
| 健康检查 | `http://127.0.0.1:8080/actuator/health` |
| 管理后台 | `http://127.0.0.1:8081` |
| H5 用户端 | `http://127.0.0.1:8082` |
| SillyTavern | `http://127.0.0.1:8000` |

### 4. 默认说明

- 示例后台账号是 `admin / admin123`
- 这个账号只用于本地体验
- 准备公开部署前，必须换掉管理员密码和各类密钥

### 5. 常用检查命令

```bash
docker compose ps
docker compose logs -f backend
```

## 方式二：本地开发

这适合你自己改代码时使用。

### 1. 准备基础依赖

先准备：

- MySQL 8
- Redis 7

说明：

- 数据库建议使用 `utf8mb4`
- 后端首次启动时会自动执行 Flyway 迁移脚本建表

可以参考根目录 `.env.example` 创建数据库：

```text
MYSQL_DATABASE=silly_app
MYSQL_USER=silly
MYSQL_PASSWORD=change-this-db-password
```

### 2. 启动 SillyTavern 组件

```bash
cd deploy
cp .env.example .env
docker compose --env-file .env up -d sillytavern
```

启动后默认地址：

```text
http://127.0.0.1:8000
```

### 3. 启动后端

PowerShell 示例：

```powershell
$env:APP_AUTH_SECRET="replace-with-long-random-secret"
$env:APP_RUOYI_ADMIN_USERNAME="admin"
$env:APP_RUOYI_ADMIN_PASSWORD="{bcrypt}replace-with-bcrypt-hash"
$env:APP_RUOYI_JWT_SECRET="replace-with-long-random-jwt-secret"
$env:APP_CORS_ALLOWED_ORIGIN_PATTERNS="http://localhost:*,http://127.0.0.1:*"
$env:SPRING_DATASOURCE_URL="jdbc:mysql://localhost:3306/silly_app?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai"
$env:SPRING_DATASOURCE_USERNAME="silly"
$env:SPRING_DATASOURCE_PASSWORD="change-this-db-password"
$env:SPRING_DATA_REDIS_HOST="localhost"
$env:SPRING_DATA_REDIS_PORT="6379"
$env:SPRING_DATA_REDIS_PASSWORD=""
$env:SILLYTAVERN_BASE_URL="http://127.0.0.1:8000"
$env:SILLYTAVERN_PUBLIC_BASE_URL="http://127.0.0.1:8000"
$env:SILLYTAVERN_API_KEY=""
```

启动：

```bash
cd backend
./mvnw spring-boot:run
```

Windows：

```powershell
cd backend
.\mvnw.cmd spring-boot:run
```

启动后访问：

```text
http://127.0.0.1:8080/actuator/health
```

### 4. 启动管理后台

```bash
cd admin-web
npm install
npm run dev
```

默认开发地址通常是：

```text
http://127.0.0.1:5173
```

后端地址可在 `admin-web/.env.development` 中配置：

```text
VITE_BACKEND_URL=http://127.0.0.1:8080
```

### 5. 启动 H5 用户端

```bash
cd h5-web
npm install
npm run dev:h5
```

说明：

- H5 本地请求地址主要在 `h5-web/common/api.js`
- 启动后以终端输出的访问地址为准
- 如果手机要访问开发机，请把 `127.0.0.1` 改成你电脑的局域网 IP

## 常见问题

### 后端连不上 MySQL

检查：

- MySQL 是否已经启动
- 数据库名、用户名、密码是否一致
- JDBC 地址是否正确

### 后端连不上 Redis

检查：

- Redis 是否已经启动
- `SPRING_DATA_REDIS_HOST`、`SPRING_DATA_REDIS_PORT`、`SPRING_DATA_REDIS_PASSWORD` 是否正确

### 聊天功能不正常

检查：

- SillyTavern 是否已经启动
- `SILLYTAVERN_BASE_URL` 是否可被后端访问
- `SILLYTAVERN_PUBLIC_BASE_URL` 是否可被浏览器访问
- 如果启用了 API Key，确认 `SILLYTAVERN_API_KEY` 已正确填写

### 管理后台或 H5 请求失败

检查：

- 后端是否已经启动
- 前端请求地址是否配对
- `APP_CORS_ALLOWED_ORIGIN_PATTERNS` 是否包含当前前端地址
