# 后端说明

后端位于 `backend/`，是 Spring Boot 应用，负责用户认证、角色、聊天、会话、记忆、SillyTavern 集成、社区、支付、工单、后台管理和运营配置。

## 基础信息

| 项目 | 内容 |
| --- | --- |
| 入口类 | `com.example.sillyspringboot.SillySpringbootApplication` |
| Java 版本 | 17 |
| Spring Boot | 4.0.5 |
| 构建工具 | Maven Wrapper |
| 默认端口 | `8080` |
| 数据库 | MySQL |
| 缓存/并发/限流 | Redis |
| 数据迁移 | Flyway |
| ORM | MyBatis + XML Mapper |

## 数据库迁移

Flyway 迁移位于：

```text
backend/src/main/resources/db/migration
```

项目包含从基础表到角色、会话、支付、工单、模型路由、BYOK、语音、社区、私聊、上传资产等多阶段迁移。启动后端时 Flyway 会自动执行未应用的迁移。

开发注意：

- 新表和字段通过新增 `Vxx__description.sql` 管理。
- 不要修改已经发布并被环境执行过的历史迁移。
- 新增 Mapper 时同步补充 Java Mapper 接口和 XML。
- 生产数据库升级前先在备份库或测试库验证迁移。

## 主要 API 前缀

| 前缀 | 用途 |
| --- | --- |
| `/api/app/auth` | H5/API 登录、注册、Telegram 登录绑定、当前用户。 |
| `/api/app/characters` | App 侧角色详情和角色导入。 |
| `/api/app/discover` | 发现页信息流。 |
| `/api/app/chat` | 新版聊天流、停止、图片上传、继续、重生成、swipe。 |

| `/api/v1/store` | 商品、订单、支付。 |
| `/api/v1/support` | 工单、回复、举报和上传。 |

| `/api/social-chat` | 用户私聊 HTTP API。 |

## 聊天链路

典型聊天流程：

1. H5 创建或读取会话。
2. H5 调用 `/api/v1/tavern/chat/stream` 或新版 `/api/app/chat/stream`。
3. 后端校验用户、设备、权益、并发和限流。
4. 后端组装角色设定、历史消息、世界书、用户人设和长期记忆。
5. 后端通过 SillyTavern 适配层或用户 BYOK 配置调用模型。
6. 后端使用 SSE 推送 `delta`、`done` 或 `error` 事件。
7. 消息、任务、审计、统计和记忆刷新按配置落库。

## 对话记忆

对话记忆由 `conversation/` 模块维护。系统会从近期聊天中抽取稳定事实，生成会话级长期记忆条目，并在后续生成时注入上下文。

常见记忆类型包括：

- 身份和称呼
- 关系
- 偏好
- 承诺
- 重要事件
- 场景设定
- 边界和禁忌

普通寒暄、一次性情绪和无意义填充不会作为长期记忆保存。

## WebSocket

私聊和实时通知相关能力位于 `humanchat/ws`。WebSocket 安全策略由以下配置控制：

```text
APP_WS_ALLOWED_ORIGIN_PATTERNS
APP_WS_ALLOW_MISSING_ORIGIN
APP_WS_REQUIRE_HANDSHAKE_TOKEN
```

生产环境建议显式配置允许的 Origin，并启用握手 token 校验。
