# 部署说明

部署模板位于 `deploy/`，用于提供一个公开、通用、无私有信息的 Docker Compose 示例。

## 文件说明

```text
deploy/
  .env.example        部署环境变量样例
  docker-compose.yml  Compose 服务编排
  st/config.yaml      SillyTavern 容器配置
  README.md           部署模板说明
```

## Compose 服务

| 服务 | 镜像/构建 | 说明 |
| --- | --- | --- |
| `mysql` | `mysql:8.0` | 业务数据库，使用 `utf8mb4`。 |
| `redis` | `redis:7-alpine` | 缓存、限流、并发状态。 |
| `sillytavern` | `ghcr.io/sillytavern/sillytavern:latest` | SillyTavern 服务。 |
| `backend` | 从 `../backend` 构建 | Spring Boot API。 |
| `admin-web` | 从 `../admin-web` 构建 | 管理后台 Nginx 静态服务。 |
| `h5-web` | 从 `../h5-web` 构建 | 自动构建 H5 并用 Nginx 提供静态服务。 |

默认端口：

| 服务 | 端口 |
| --- | --- |
| 后端 | `8080` |
| 管理端 | `8081` |
| H5 | `8082` |
| SillyTavern | `8000` |

## 部署步骤

1. 复制环境变量。

```bash
cd deploy
cp .env.example .env
```

2. 修改 `.env`。

必须替换所有 `change-this-*` 和占位密钥。重点检查：

```text
MYSQL_ROOT_PASSWORD
MYSQL_PASSWORD
APP_AUTH_SECRET
APP_RUOYI_ADMIN_PASSWORD
APP_RUOYI_JWT_SECRET
APP_CORS_ALLOWED_ORIGIN_PATTERNS
SILLYTAVERN_PUBLIC_BASE_URL
SILLYTAVERN_API_KEY
```

3. 启动。

```bash
docker compose --env-file .env up -d --build
```

该命令会自动构建并启动后端、管理端、H5、MySQL、Redis 和 SillyTavern。

4. 查看状态。

```bash
docker compose ps
docker compose logs -f backend
```

## 数据卷

Compose 声明了以下卷：

| 卷 | 用途 |
| --- | --- |
| `mysql-data` | MySQL 数据。 |
| `redis-data` | Redis AOF 数据。 |
| `uploads` | 后端上传文件。 |
| `st-asset-cache` | SillyTavern 资源缓存。 |
| `st-config` | SillyTavern 配置。 |
| `st-data` | SillyTavern 数据目录。 |
| `st-plugins` | SillyTavern 插件。 |
| `st-extensions` | SillyTavern 第三方扩展。 |

生产环境需要为这些卷制定备份策略。

## SillyTavern 接入

当前 Compose 会启动 `sillytavern` 服务，后端在容器网络内通过以下地址访问：

```text
http://sillytavern:8000
```

`SILLYTAVERN_PUBLIC_BASE_URL` 用于需要返回给浏览器访问的公共地址；本地默认可以使用 `http://127.0.0.1:8000`。如启用 SillyTavern API Key，请在 `.env` 中配置 `SILLYTAVERN_API_KEY`。

## 反向代理建议

生产部署通常会在 Compose 外层放置 Nginx、Caddy、Traefik 或云厂商负载均衡。建议：

- H5 和 API 使用同一主域名下的不同路径，降低跨域复杂度。
- WebSocket 路径保留 `Upgrade` 和 `Connection` 头。
- SSE 聊天接口关闭代理缓冲或调大超时。
- 上传接口配置合适的 body size 限制。
- 静态上传资源建议通过受控路径暴露，不要直接暴露容器内部目录。

## 生产安全检查

上线前确认：

- `.env` 不在 Git 仓库。
- 所有默认密钥已替换。
- 管理员初始密码已修改并使用 bcrypt。
- 数据库和 Redis 不直接暴露公网。
- CORS 和 WebSocket Origin 只允许可信域名。
- Mock 支付不在生产开放，或确认 `APP_PAYMENT_MOCK_ALLOW_IN_PROD=false`。
- SillyTavern API Key、支付私钥、Telegram token 已安全注入。
- 已开启数据库和上传文件备份。

## 常见问题

### H5 容器构建失败

查看 H5 构建日志：

```bash
docker compose logs -f h5-web
```

也可以在本地先执行 `cd h5-web && npm run build:h5` 验证 CLI 构建。

### 后端连不上 SillyTavern

确认 `sillytavern` 服务健康，并查看日志：

```bash
docker compose ps sillytavern
docker compose logs -f sillytavern
```

### 数据库迁移失败

查看后端日志：

```bash
docker compose logs -f backend
```

生产环境不要直接修改已执行过的 Flyway 历史迁移，应新增迁移修复。
