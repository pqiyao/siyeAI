# 开发维护

本文档面向准备二次开发、长期维护或把本项目改造成自己服务的开发者。

## 推荐开发分工

| 模块 | 主要职责 | 常见改动 |
| --- | --- | --- |
| `h5-web/` | 用户端体验 | 角色展示、聊天交互、个人中心、支付入口、工单反馈。 |
| `admin-web/` | 运营后台 | 用户管理、角色审核、订单权益、公告、工单、插画管理。 |
| `backend/` | 业务后端 | 鉴权、用户、角色、会话、消息、权益、订单、模型配置、上传。 |
| `integrations/` | AI 运行时集成 | SillyTavern、世界书、记忆增强、上下文处理。 |
| `deploy/` | 部署交付 | Docker Compose、Nginx、环境变量、端口和容器编排。 |

## 日常开发流程

1. 先阅读 [01-项目概览.md](01-项目概览.md)，确认业务边界。
2. 根据改动模块阅读对应文档：后端看 [04-后端说明.md](04-后端说明.md)，前端看 [05-前端说明.md](05-前端说明.md)。
3. 新增配置时，同步更新 `.env.example`、`deploy/.env.example` 和文档。
4. 新增数据库字段或表时，使用 Flyway 迁移脚本，不依赖手工改库。
5. 改动接口字段时，同时检查 H5、后台和相关说明文档。
6. 提交前确认没有提交密钥、构建产物、上传文件和真实用户数据。

## 本地验证建议

后端：

```bash
cd backend
./mvnw spring-boot:run
```

后台：

```bash
cd admin-web
npm install
npm run dev
```

H5：

```bash
cd h5-web
npm install
npm run dev:h5
```

全栈 Docker 预览：

```bash
cd deploy
cp .env.example .env
docker compose --env-file .env up -d --build
```

## 数据库维护

数据库结构以 `backend/src/main/resources/db/migration/` 中的 Flyway 迁移为准。

维护建议：

- 新表、新字段、新索引通过新迁移脚本追加。
- 不要修改已经发布过的迁移脚本，避免旧环境升级失败。
- 迁移脚本命名保持版本号递增。
- 重要业务表改动后，同步更新相关说明文档。

## 敏感配置维护

敏感配置不进入 Git：

- 数据库密码。
- Redis 密码。
- JWT 或业务签名密钥。
- 模型供应商 Key。
- 支付密钥和证书。
- 生产后台账号密码。

示例文件只保留占位符或本地演示值，生产环境通过服务器环境变量、容器环境变量或密钥管理系统注入。
